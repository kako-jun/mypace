# 404 Page

存在しないルートにアクセスした場合のフォールバックページ。

## Overview

無効なURLにアクセスした場合、真っ白な画面ではなく404ページを表示する。

## Behavior

1. 存在しないパスにアクセス（例: `/invalid-path`）
2. 「404 - Page not found」と「Go to Home」リンクが表示される
3. ホームへのリンクをクリックするとトップページに戻る

## Implementation

### Route

`App.tsx` でフォールバックルートを設定:

```typescript
<Route path="*" element={<NotFoundPage />} />
```

### Component

`pages/NotFoundPage.tsx`:

```typescript
export function NotFoundPage() {
  return (
    <div className="container" style={{ textAlign: 'center', paddingTop: '4rem' }}>
      <h1 style={{ fontSize: '3rem', marginBottom: '1rem' }}>404</h1>
      <p style={{ marginBottom: '2rem', opacity: 0.7 }}>Page not found</p>
      <Link to="/" className="text-button">
        Go to Home
      </Link>
    </div>
  )
}
```

## Use Cases

- 古いバージョンで新機能のURLにアクセス（例: `/intent/post`）
- 存在しないユーザーページ以外の無効なパス
- タイプミスによるURL
